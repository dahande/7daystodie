Nemesis System (detailed):

- Persistent world roster with controlled population:
	+ Nemesis pool runs with dynamic bounds.
	+ Rank distribution target is configurable per tier, with hard cap priority.

- Persistent identity and progression per nemesis:
	+ Each nemesis stores unique identity data (name, title, rank, entity class/tier, home coordinates, last known entity id).
	+ Combat and simulation state is persisted (virtual HP/Attack/Defense/Speed, win/loss streaks, kill/death counters).
	+ Title source is tracked (for example rank- or glory-driven title assignment).

- Death lifecycle and slot recycling:
	+ Permanent death threshold is rolled per nemesis between configurable limits.
	+ After death/despawn, respawn is blocked by cooldown.
	+ Empty slots are not instantly replaced.
	+ Home-zone proximity spawns are rate-limited.

- Grudge and revenge logic:
	+ Nemesis can lock a grudge target and evaluate attacks on timed checks.
	+ Repeated revenge pressure is throttled by per pair cooldown.
	+ Freshly respawned players get anti-chain protection.

- Family vengeance layer:
	+ Nemesis kills can trigger untracked same-family retaliation near the killer.
	+ Killing related non-nemesis AEC bosses can also trigger linked nemesis grudges.

- Dynamic hierarchy conflicts:
	+ Internal power struggles trigger on interval windows.
	+ Territory war encounter windows are timeboxed.
	+ Related ally intervention can occur during active war.
	+ Ally support applies simulation boosts.

- Territory migration behavior:
	+ Nemesis can relocate home territory on long-form intervals.

- State continuity:
	+ Nemesis runtime state persists through JSON state files to keep world-level continuity between sessions.

Additional Systems Breakdown:

- World Events framework:
	+ Every biome runs its own random event timer.
	+ Events can chain temporary surge windows with targeted enemy families.
	+ Supported event themes include Mushroom, Archer, Headless, Ghost (night), Singerie, Druid and Doomlord invasions.
	+ Some events can trigger large horde-style pressure spikes and biome-wide reinforcement behavior.

- Wilderness ecosystem:
	+ AEC scaled boss/minion ecosystem is focused on wilderness pressure loops.
	+ Dynamic wilderness spawn pressure is tied to biome systems and can escalate during active event windows.

- Quest layer (T05-T15):
	+ Quest availability is tied to player game stage and progression state.
	+ Dialog flow adapts to progression, with selected lines hidden until the proper progression threshold is reached.
	+ Quest wilderness overrides are designed to preserve tier pressure while keeping wilderness pacing aligned with AEC scaling.

- Pool and runtime modules:
	+ AI-driven pool/no-quest behavior is integrated as a dedicated runtime module.
	+ Runtime modules are designed to report startup state clearly (loaded/settings applied) to ease server diagnostics.

- Passive progression support:
	+ Additional passive-skill and utility tabs are integrated for AEC progression support.
	+ The system includes progression hooks for boss/minion kill loops and higher-tier reward paths.

- Nemesis gameplay loop (player-facing behavior):
	+ Nemesis attacks can occur near players after runtime scheduling windows.
	+ Nemesis can challenge other nemesis during territory conflicts.
	+ Nemesis can gain/lose rank over time depending on victories and defeats.
	+ Defeated nemesis are handled through cooldown/permanent-death rules, with roster refill when required.
	+ Grudge behavior can bias aggression toward players who previously defeated that nemesis.

- Heatmap gameplay loop:
	+ Heatmap reacts to biome kill activity and increases hazard pressure dynamically.
	+ Hazard level can influence local wilderness difficulty and boss pressure.
	+ Heatmap decay and escalation timing are governed by runtime rules to avoid constant oscillation.
	+ Low/High game stage handling is aligned to dedicated merged spawn rule paths for consistent tier selection.

- Event rules and challenge support:
	+ Event-rule runtime settings are loaded from the AEC ENDGAME module configuration path.
	+ Challenge tabs were expanded to better expose high-tier objective progression.

- Benches and crafting progression:
	+ Head Converter bench supports conversion of AEC AIO BOSS heads into AEC ENDGAME heads.
	+ The AEC Extreme Workstation progression includes large modifier/mod crafting depth (high-volume endgame itemization).
